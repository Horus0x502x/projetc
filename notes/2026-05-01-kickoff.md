## Addendum — Findings phase 1

Cf. `recon/00-findings-clefs.md` pour la synthèse complète. Highlights :

1. **`iw4v-plus.dll` est packé Themida** — toutes les sections sont chiffrées
   (entropie ≥ 7.93), section `.themida` + `.boot` typiques. Reverse statique direct
   = enfer. Ignoré comme cible d'apprentissage. À documenter seulement.
2. **`iw4v.dll` est un fork local** d'un user `scfge` (PDB leak :
   `C:\Users\scfge\source\repos\iw4x-client-eliott\iw4x-client\...`). Excellent
   candidat pour un exercice de **diffing** (`iw4v.dll` vs upstream `iw4x.dll`).
3. **`iw4mp.exe` / `iw4sp.exe`** confirmés binaires officiels Infinity Ward 2009
   (PDB leak `c:\trees\iw4-pc\...`, URL `iwnet.infinityward.com`). Strippés
   commercialement par CEG sur `iw4sp`.
4. **Le launcher Linux** non-strippé est notre **point d'entrée pédagogique #1** —
   31 871 symboles, DWARF complet, statiquement lié (curl + JSON5 + SQLite embarqués).
5. **Pas de packing** détecté sur les autres binaires (entropy `.text` ≈ 6.7 partout —
   typique de code x86 non compressé).
6. **Crash dump du jour** (`iw4x-r5039-20260501001438.dmp`) → matériau pour phase 3
   (analyse de minidump avec dbghelp).
